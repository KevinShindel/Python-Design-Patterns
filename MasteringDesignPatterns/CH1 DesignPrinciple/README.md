# The Four main design principles are:
1. Encapsulate what varies
2. Favor composition over inheritance
3. Program to an interface, not an implementation
4. Loose coupling


## 1. Encapsulate what varies
> isolate the parts of the code that are likely to change, and separate them from the parts that are not likely to change.
This way, when changes occur, they will only affect a small portion of the code, making it easier to maintain and update.

Benefits: Maintenance, Reusability, Flexibility
Techniques: Polymorphism, Getters/Setters, Strategy Pattern
